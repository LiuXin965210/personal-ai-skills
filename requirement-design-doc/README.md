# Requirement Design Doc

根据 PRD、设计稿链接或文本生成 Markdown 技术设计文档，可结合后端工程确认接口、Service、数据模型、消息和 SQL 变更。

## 适用场景

- 从产品需求生成后端技术设计稿。
- 根据指定 PDF 或 Markdown 模板生成文档。
- 结合现有工程分析改动点和复用能力。
- 设计接口、Kafka、Service、数据模型和增量脚本。

## 使用前准备

- 提供需求链接、正文或导出文档。
- 可选：后端工程名称或本地路径。
- 可选：自定义 PDF、Markdown 或文本模板。
- 可选：指定输出文件名和目录。

未提供模板时会自动使用技能内置模板，不需要额外准备。

## 使用方式

```text
使用 requirement-design-doc 根据下面需求生成后端设计稿，
后端工程在 /path/to/project：
<需求正文或链接>
```

使用自定义模板：

```text
按 /path/to/template.pdf 的格式，
为 <PRD 链接> 生成技术设计稿，保存到 /path/to/output.md
```

## 默认输出

文档通常包含：

- 变更记录和需求链接。
- 外围相关接口和接口设计。
- Kafka 或其他消息设计。
- Service 设计和数据模型。
- 增量变更脚本。
- 工程影响点和待确认事项。

默认在当前工作区根目录生成 Markdown 文件。

## 注意事项

- 提供工程路径时，接口、类名、字段和 SQL 会优先依据现有代码。
- 无法确认的实现细节会标记为“建议”或“待确认”，不会伪造事实。
- 需求链接需要登录时，请在浏览器中自行完成授权，不要在聊天中提供密码或 Token。
- 用户模板与内置模板冲突时，以用户本次提供的模板为准。
