# PRD Task Breakdown

读取 PRD 链接、正文或导出文档，将需求拆成连续任务项，并按给定预算汇总后端、前端和测试工时。

## 适用场景

- 将产品需求拆成研发任务清单。
- 按后端、前端、测试角色分配人天。
- 将大需求拆成单项不超过指定工时的交付项。
- 从受登录保护的 PRD 页面提取需求并拆解。

## 使用前准备

建议提供：

- PRD 链接、正文、Markdown、Word 或 PDF。
- 后端、前端、测试的总工时预算。
- 单项工时上限，未指定时默认为 5 天。
- 是否需要逐项展示角色工时，默认只展示总计。

## 使用方式

```text
使用 prd-task-breakdown 拆解下面的 PRD。
后端 10 天，前端 6 天，测试 4 天，默认格式输出：
<PRD 正文或链接>
```

要求逐项显示工时：

```text
拆解这个 PRD，后端 8 天、前端 5 天、测试 3 天，
每项标注角色工时
```

## 输出结果

默认输出连续编号的任务列表，并在末尾汇总：

```text
1. 任务描述
2. 任务描述
总计：后端X天 前端Y天 测试Z天 合计N天
```

## 注意事项

- 默认每个任务项不超过 5 天，总工时不超过 40 天。
- 角色总工时会严格匹配用户给定预算；预算缺失或冲突时会先指出。
- 默认不按模块分成多个卡片，也不在任务项下继续拆子列表。
- PRD 链接需要登录时，只会请用户在浏览器内完成授权，不会索要密码、Cookie 或 Token。
